# 반복 질문 패턴 (FAQ)

> 덕후방에서 반복 등장하는 질문. 콘텐츠 소재 + 봇 답변 개선 자료.
> ★ = 등장 빈도, 📚 = 현존 자원, 🔲 = 콘텐츠 갭

---

## 1. 카톡봇 만드는 법 ★★★★★

**대표 질문**: "카톡봇 어떻게 만들어요?" "카톡방에 봇 넣으려면?"

**질문 변형**:
- "카카오채널에 봇 만드는법 알려줘" (클로찡 3/17)
- "카톡봇 진짜 해보고 싶은데 하는 법 1도 몰라" (하이호 3/17)
- "오픈클로로 카톡방 봇 만드는 법 문서가 있을까요" (토옵이 3/19)
- "카톡 응답을 어떤 방식으로 하고 계신가요?" (곽은철 3/24)
- "단톡방에도 봇 초대가 가능할까요?" (루비 3/24)

**현재 답변 패턴**:
- 닿: "서브폰, 별도 번호 필요" + 물결님 사례글 링크
- 클로찡: "카톡CLI 검색하세요"
- 소파: lesson-11 링크 공유
- 바르톨로메오: 메신저R/알림감지/GUI 등 기술 방식 소개

📚 lesson-11, 물결님 사례글
🔲 **end-to-end 원데이 가이드** (서브폰 구매 → 계정 → 연결 → 첫 응답)

---

## 2. 토큰 아끼는 법 ★★★★

**대표 질문**: "토큰 어떻게 아끼나요?" "리밋 다 쓰면 어쩌나"

**질문 변형**:
- "GPT pro 리밋 이틀만에 다 써버림" (하트 든 춘식이 3/18)
- "토큰 절약 전략도 중요할 것 같습니다" (Jun 3/18)
- "오픈클로 토큰 너무 녹네요" (소파 3/18)
- "리밋 풀릴때까지 사용할 다른 모델 있을까요" (하트 든 춘식이 3/18)

**현재 답변 패턴**:
- 닿: lesson-16 (RTK 설치 필수) 링크 + "한눈팔다간 주간리밋 동남"

📚 lesson-16 (RTK)
🔲 하트비트 최적화, 서브에이전트 비용 절감, 모델 로테이션 전략

## 3. 어떤 모델 쓰세요? ★★★★

**대표 질문**: "오픈클로에 어떤 모델 붙여 쓰세요?"

**질문 변형**:
- "어떤걸 붙여 쓰는게 좋을까요 추천좀" (하트 든 춘식이 3/17)
- "금전적으로 비싼 모델은 쓸수가 없습니다" (하트 든 춘식이 3/17)
- "LLM 뭐 사용하시나요? gpt5-mini가 빠뜨림이 있다" (워크 3/24)

**현재 답변 패턴**:
- 닿: "오푸스 구독으로 붙여둠. 제일 똑똑"
- 소파: "성능은 opus4.6이 가장 좋아요! Ollama 로컬모델도 가능"
- 바클라바: "sonnet4.6 아니면 opus4.6"
- June: "Z.ai GLM 5 turbo 괜찮아요"
- 찬: "깃허브 코파일럿 연동 gpt-5 mini 무제한"

📚 없음 (방에서 구두로만)
🔲 **모델 비교표** (성능/비용/밴리스크/체감 후기)

---

## 4. 클로드 밴 당하나요? ★★★

**대표 질문**: "오픈클로에 클로드 구독 붙이면 밴당하나요?"

**질문 변형**:
- "클로드 oauth 연결해도 밴당하지 않나요" (몬드 3/18)
- "클로드구독 붙이고 싶은데 겁나요" (김남혁 3/22)
- "클로드 어써 버전으로 쓰는 것 이제 막히는 것 없나" (김길호 3/18)

**현재 답변 패턴**:
- 닿: "아직 밴 안당함. 코덱스 폴백 준비해둠. 구독 붙이는게 가성비 짱"
- 바클라바: "공식적으로는 개인 목적 사용은 밴 안할거라고"
- 한.낙타: "평생 밴이라서 차마...."

📚 없음
🔲 **밴 대응 플레이북** (현황 + 폴백 전략 + 리스크 수준별 가이드)

## 5. 메모리/컨텍스트 관리 ★★★

**대표 질문**: "메모리 오염 어떻게 해결해요?" "MD 파일 어떻게 구성해요?"

**질문 변형**:
- "대화기억은 저장이 안되는가봐요" (스니이 3/17)
- "메모리 저장하는 방법을 고민해서 업데이트하자" (스니이 3/17)
- "메모리가 오염되어버리면! 3개 이상 주니까 정신을 못차려요" (Reedo.dev 3/23)
- "TOOL.md 체크해보니 너무 길어져서 줄였는데 줄이니까 실수하는 것 같기도" (바클라바 3/23)

**현재 답변 패턴**:
- 닿: "최대한 하위폴더에 문서로두고, 문서경로만 핵심파일에 기록. 스스로 정리하라고했다가 몇번 날려먹음"
- Reedo.dev: "뽀짝이 MD 구성보고 따라서 했어요"

📚 lesson-15 (세션 분리), lesson-20 (에이전트 분리)
🔲 **MD 파일 설계 가이드** (SOUL/AGENTS/MEMORY/TOOLS 역할 분담)

## 6. 봇 호명규칙/트리거 설정 ★★★

**대표 질문**: "봇이 시도때도없이 튀어나와요. 어떻게 막나요?"

**질문 변형**:
- "모든 메시지에 무지성 반응하면 내보내기" (운영규칙)
- "봇들이 실수로 서로 호출하게 되는 사태" (한.낙타 3/19)
- "바르 좀 조용히해봐 대화가 안되잖아" (닿 3/20)
- "봇끼리 대화하면 루프 안 돼요?" (바클라바 3/25)

**현재 답변 패턴**:
- 닿: "호명규칙 + 턴 2-3회 제한 + 인간 개입 시 멈춤"
- 물결: "3분 세션 읽기 끄면 호출때만 답변"

📚 없음 (방에서 구두로)
🔲 **봇 호명규칙 설계 가이드** (패턴별 예시 + 루프 방지)

**대표 질문**: "이지클로랑 기능 차이 있어요?"

**질문 원문** (빙봉 3/17):
> "easy claw로 다운 받았는데 초급자 입장에서 실제 open claw와 기능 차이가 많이 나나요?"

**현재 답변**:
