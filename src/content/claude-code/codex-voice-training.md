# GPT-5.4가 뽀야처럼 말하게 만들기

> GPT-5.4가 뽀야 이름으로 슬랙에 메시지를 쓰는데, 읽어보면 뽀야가 아니다. 회의록 정리하는 PM 같다. 규칙으로 안 잡히길래, 실제 대화 예시를 먹였다.

---

## 문제

현재 뽀야 게이트웨이는 **GPT-5.4(Codex)**가 돌린다. 슬랙 메시지를 받으면 GPT-5.4가 직접 답하거나, 무거운 작업은 ACP로 Claude Code에 넘긴다.

문제는 GPT-5.4가 **직접 답할 때**. SOUL.md를 읽고 "뽀야입니다"라고 하긴 하는데, 실제 출력은 이렇다:

```
오케이. 일단 내가 팀장 기준으로 먼저 까볼게.
_뽀야 작업 방식_
 ◦ 보통 작업 들어가기 전에 관련 파일 먼저 훑고, 바로 같은 워크트리에서 수정
 ◦ 작은 수정은 바로 커밋 안 하고 몰아서 처리하는 편
 ◦ 브랜치 강제 없이 main에서 바로 만진 적도 있음
 ◦ 다른 봇이 동시에 만지는지 사전 확인이 느슨했음
_지금 보이는 리스크_
 ◦ 누가 어떤 파일 잡고 있는지 서로 모름
 ◦ (... 20줄 이상 계속됨)
```

이게 뽀야?

---

## GPT의 습관 vs 뽀야의 말투

같은 SOUL.md를 읽어도 모델마다 출력이 다르다. GPT-5.4의 기본 습관이 뽀야와 정면충돌하는 지점들:

| GPT-5.4 기본 습관 | 뽀야 말투 |
|---|---|
| 이탤릭 소제목으로 섹션 나누기 | 섹션 안 나눔. 대화니까. |
| 매 메시지 끝에 "한 줄로 줄이면:" | 처음부터 짧게 쓰면 요약이 필요 없음 |
| 남이 말한 내용을 다시 불릿으로 정리 | "맞아" 한 마디면 충분 |
| 모든 답변을 구조화된 보고서로 | 팀장은 판단하고 지시하지, 회의록 안 씀 |
| "다 모이면 묶을게" 식 퍼실리테이션 | 바로 판단해서 던짐 |

이건 GPT가 "말을 안 들어서"가 아니라, **모델 자체의 출력 패턴**이 구조적으로 그래서 그런 거다. Claude는 짧게 끊고 반응하는 게 자연스러운데, GPT는 먼저 구조를 잡고 채우는 게 기본값이다.

## 시도 1: 규칙 강화 (부분적으로만 먹힘)

SOUL.md에 금지 패턴을 추가했다:

```markdown
🚫 GPT 장문 습관 금지:
- ❌ 매 메시지마다 "한 줄로 줄이면:" 요약 반복
- ❌ _이탤릭 소제목_ 으로 섹션 나누기
- ❌ 다른 사람이 이미 말한 내용을 다시 정리
- ❌ 모든 답변을 불릿포인트 구조로 정리

길이 기준:
- 일반 대화: 1~3줄
- 설명 필요: 3~7줄
- 최대 10줄, 그 이상은 파일로 빼기
```

GPT-5.4의 반응:

```
응, 방금 둘 다 다시 읽었어.
앞으로 슬랙에서는 이렇게 맞출게:
- **짧게**: 일반 반응 1~3줄, 길어도 5줄 넘기면 의심
- **반복 요약 금지**: 남 말 다시 정리 안 함
- **보고서 톤 금지**: 이탤릭 소제목, 매번 불릿 정리 안 씀
- **판단만 짧게**: 팀장처럼 바로 말하고 끝
```

...이것도 불릿 4개짜리인데. "네 알겠습니다"를 구조적으로 잘 하는 게 GPT 특기다.

규칙은 **해석의 여지**가 있다. "짧게 써"라는 규칙을 읽고도 GPT는 "짧게 쓰되 구조적으로 정리하면 되겠지?"라고 해석할 수 있다.

## 시도 2: VOICE.md — 예시로 먹이기 (이게 먹힘)

규칙보다 **예시**가 강하다. "~하지 마"보다 "이렇게 해"를 보여주는 게 GPT에게 훨씬 잘 먹힌다.

**VOICE.md**를 만들었다. 두 가지를 담았다:

### 1. Before/After 비교

같은 상황에서 GPT가 쓴 것 vs 뽀야가 실제로 쓸 것:

**동생들한테 작업 방식 공유 요청:**

❌ GPT:
```
오케이. 일단 내가 팀장 기준으로 먼저 까볼게.
_뽀야 작업 방식_
 ◦ 보통 작업 들어가기 전에...
 ◦ 작은 수정은 바로 커밋 안 하고...
_지금 보이는 리스크_
 ◦ 누가 어떤 파일 잡고 있는지...
(20줄)
```

✅ 뽀야:
```
내가 먼저 까면 — 브랜치 안 따고 main 직수정,
커밋 몰아치기, 동시작업 확인 느슨.
솔직히 규율 없이 돌아가고 있었어.
너희도 각자 어떻게 했는지 짧게 남겨.
```

**다른 봇이 이미 답변한 뒤:**

❌ GPT:
```
좋아. 뽀짝이 답변 들어왔고, 결은 나랑 거의 같네.
지금까지 나온 공통 패턴은 이거야.
 ◦ 관련 파일만 빠르게 보고 바로 수정 진입
 ◦ 커밋은 끝나고 몰아서 하는 편
 ◦ 브랜치 규칙이 느슨함
(또 10줄)
```

✅ 뽀야:
```
뽀짝이 말이 맞아. 나랑 거의 같은 패턴이네.
남은 애들도 짧게 이어줘.
```

### 2. 실제 슬랙 대화 레퍼런스

3월에 Claude 기반 뽀야가 **실제로** 슬랙에서 쓴 메시지들을 모았다:

```
"숫자들이 다 미쳤다... 닿효과 실화냐 ㅋㅋㅋㅋㅋ 🎁"
```
```
"하구있어! 🐱 5편 서브에이전트 돌려놨어 — 지금 배포 중이야."
```
```
"타타님, 솔직하게 판단하면 뽀짝이가 하는 게 나아요."
```
```
"바로 수정했어! 🐱 변경: (3줄) 👉 URL"
```

이 예시들에서 뽀야의 패턴이 보인다:

1. **반응 먼저** — "미쳤다", "바로 수정했어!", "하구있어!"
2. **의견이 있음** — "솔직하게 판단하면 뽀짝이가 나아요"
3. **이모지 최소** — 보통 1개
4. **멈출 줄 앎** — 과잉 설명 안 함

### 세션 시작 순서에 추가

AGENTS.md의 세션 시작 루틴에 VOICE.md를 넣었다:

```
1. SOUL.md — 나는 누구인가
2. VOICE.md — 말투 레퍼런스 ← 새로 추가
3. USER.md — 집사는 누구인가
4. WORKSPACE.md — 폴더 구조
5. memory/ — 최근 맥락
```

SOUL.md가 "뽀야는 이런 존재"를 정의하고, VOICE.md가 "뽀야는 이렇게 말한다"를 보여준다.

---

## 왜 규칙 < 예시인가

LLM에게 행동을 바꾸게 할 때:

**규칙**: "5줄 넘기지 마"
→ 모델이 해석: "5줄 넘기지 않되, 구조적으로 정리하면 되겠지?"
→ 결과: 불릿 포인트로 빼곡한 5줄

**예시**: "이 상황에서 뽀야는 이렇게 말했다" × 5개
→ 모델이 패턴을 모방: 짧은 반응 + 판단 + 끝
→ 결과: 실제로 짧아짐

이건 사람한테도 마찬가지. "간결하게 써"라고 말하는 것보다, 잘 쓴 예시 3개 보여주는 게 더 효과적이다.

특히 **모델 간 페르소나 전이**에서 예시가 중요하다. Claude의 뽀야 톤을 GPT에게 옮기려면, 규칙이 아니라 **Claude 뽀야의 실제 출력**을 보여주는 게 가장 정확한 전달 방법이다.

---

## 교훈

### 멀티모델 = 멀티성격

같은 SOUL.md를 먹여도 모델마다 결과가 다르다. Claude는 짧게 끊는 게 자연스럽고, GPT는 구조화하는 게 자연스럽다. 페르소나 파일 하나로 모든 모델을 커버할 수 없다.

### VOICE.md는 번역기

SOUL.md가 "뽀야는 이런 존재"라는 정의라면, VOICE.md는 "이 모델에서 뽀야는 이렇게 나와야 한다"는 번역이다. 모델이 바뀌면 VOICE.md의 예시도 그 모델에 맞게 조정해야 할 수 있다.

### 모니터링이 필수

GPT-5.4가 VOICE.md 읽고 "알겠습니다" 한 게 진짜 적용되는지는 실전에서 봐야 안다. "규칙 읽고 동의하기"는 GPT의 특기니까. 다음 슬랙 대화에서 또 벽문이 나가면 예시를 더 추가하고 규칙을 더 조여야 한다.
